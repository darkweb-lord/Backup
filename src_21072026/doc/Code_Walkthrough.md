# D_SEM Firmware Architecture Walkthrough
**Project:** Smart Energy Meter (D_SEM)  
**Company:** SiliconShift  
**MCU:** PIC24FV32KA304  

## Overview: Layered Architecture
This firmware utilizes a highly modular Layered Architecture. It isolates hardware protocols from application logic, making the system scalable, robust, and highly portable.

### 1. Configuration Layer
* **Files:** `system.c`, `pin_manager.c`
* **Mechanics:** Configures the FRCPLL oscillator to 32MHz (yielding an FCY of 16MHz). It explicitly sets the `I2C1SEL` pragma to `SEC` to route the I2C peripheral to the alternate pins (ASCL1/ASDA1). The Pin Manager handles latch clearing and digital switch pin mapping.

### 2. Hardware Abstraction Layer (HAL)
* **Files:** `hal_i2c.c`, `hal_spi.c`
* **Mechanics:** Operates entirely agnostically of connected devices. 
  * The **I2C HAL** handles 100kHz standard transmission, exposing both raw block writes (used for LCD manipulation) and memory-mapped register writes (used for the RTC/EEPROM). 
  * **Bus Recovery (`HAL_I2C1_BusRecovery`):** Runs automatically during I2C initialization. If a slave device is stuck holding the SDA line LOW due to an unexpected MCU reset, this function temporarily disables the I2C peripheral and manually pulses the SCL pin via GPIO up to 9 times to clear the deadlock.
  * The **SPI HAL** manages full-duplex transfers to interact with the AFE.

### 3. Driver Layer
* **Files:** `driver_lcd.c`, `driver_rtcc.c`, `driver_atsense.c`, `driver_button.c`, `driver_eeprom.c`
* **Mechanics:** 
  * **LCD (AiP31068):** A Native I2C LCD driver. It uses 8-bit mode natively over I2C, drastically improving speed over standard backpacks. Features a variadic `LCD_PRINTF` function using `vsnprintf` to format text, integers, and floats directly to the screen safely without buffer overflows.
  * **RTC (MCP79412):** Parses the binary-coded decimal (BCD) values from address `0x6F`. It fully implements Bit 6 (12/24H mode) and Bit 5 (AM/PM) logic for data ingestion and translation.
  * **EEPROM (Dual Architecture):** 
    * **RTC EEPROM (`0x57`):** Uses the external MCP79412's 128-byte EEPROM for high-frequency data logging (like cumulative kWh). It has 1,000,000 write cycles and does *not* pause the CPU during writes.
    * **MCU EEPROM (Internal):** Uses the PIC24's internal 2KB EEPROM for static factory configurations (like Calibration Constants). Writes pause the CPU, making it unsafe for active metering, but ideal for set-and-forget data.
  * **Metrology:** Triggers off an external interrupt (`INT1`) generated by the ATSENSE301H Data Ready (`DRDY`) pin. It grabs raw 24-bit samples over SPI and applies sign-extension to handle negative voltage swings.
  * **Button Matrix:** Uses a non-blocking tracking timer rather than `__delay_ms()` to debounce keys, identifying short presses and simultaneous long presses without freezing UI rendering or missing critical interrupts.

### 4. Application Layer
* **Files:** `app_state_machine.c`, `main.c`
* **Mechanics:**
  * Replaces monolithic switch statements with a **Function Pointer Array** (`StateTable[]`). The main loop (`App_RunStateMachine`) dynamically executes whatever function the `current_state` points to. Adding a screen requires appending just one function to the array.
  * Incorporates a non-blocking scrolling text banner (`scroll_tick`) and safe time-format conversion math to transition between 12-Hour and 24-Hour displays without scrambling the RTC's registers.
